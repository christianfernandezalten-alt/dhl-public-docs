## 🔢 Versioning Scheme

This project utilizes an **internal, date- and sprint-based versioning scheme** designed for rapid identification of the specific development cycle associated with any given build.

### Format Structure: `YYMM.S.P`

| Component | Description | Example |
| :--- | :--- | :--- |
| **YYMM** | **Year and Month.** Represents the year (YY) and month (MM) when the development cycle started. | `2509` (September 2025) |
| **S** | **Sprint Identifier.** Indicates the half of the month: | **`1`**: First half of the month (Sprint A)<br>**`2`**: Second half of the month (Sprint B) |
| **P** | **Patch/Progressive Sequence.** A sequential number **starting at 0** that increments for subsequent builds, hotfixes, or patches within that specific sprint cycle. | `0` (Initial release), `1`, `2`, `3`, etc. |

### Examples

- **`2509.1.0`**: The **initial release** for **Sprint 1** (first half) of September 2025.
- **`2509.1.1`**: The **first patch/hotfix** following the initial release of Sprint 1, September 2025.
- **`2509.2.5`**: The **sixth release** (patch sequence 5) for **Sprint 2** (second half) of September 2025.
